# Interpreting Results

Understand the metrics and charts generated by Tressi.

## Key Metrics

### Latency (Response Time)

Tressi tracks how long each request takes. We focus on percentiles:

- **P50 (Median)**: 50% of requests were faster than this.
- **P95**: 95% of requests were faster than this. A good indicator of "worst-case" performance for most users.
- **P99**: 99% of requests were faster than this. Useful for identifying tail latency issues.

### Throughput (RPS)

The number of successful requests per second.

### Success Rate

The ratio of successful (2xx) responses to total requests.

## Charts

### Performance Over Time

Shows how RPS and Latency fluctuate during the test duration.

### Latency Distribution

A histogram showing the frequency of different response times.
